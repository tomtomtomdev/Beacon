- **2026-07-09** — **Slice 9 built (LLM fallback classifier); decisions during the build:** (1) **The LLM is a transparent upgrader behind the one `Classifier` port, not a pipeline branch.** `TieredClassifier(heuristic, llm, budget)` *is* a `Classifier`, so `ingest_source`/`ingest_all`/`ingest_companyless_source`/`backfill` are untouched — they still take a single `classifier` and never learn the LLM exists (golden rule: new capability, zero changes to application/domain). Wired in only at the composition roots via `make_classifier` (mirrors `make_notifier`): heuristic-only until `BEACON_ANTHROPIC_API_KEY` is set, else the tiered classifier. (2) **The ambiguity gate is a single pure predicate `Classification.is_ambiguous` (= empty category set)** in the domain (mirrors `TelegramConfig.is_configured`), so "ambiguous" is defined in exactly one place ("explicit ambiguity signal, not a magic number sprinkled around"). **Level-UNSPECIFIED alone is NOT ambiguous** — `unspecified` is honest (SPEC §6) and not worth a paid call once the category is known; only empty *categories* trigger the LLM. (3) **The LLM is a strict upgrader, never a dependency:** it's consulted only when `is_ambiguous` AND the budget allows, and **any** failure (bad JSON, HTTP/transport error, refusal) is caught and degrades silently to the heuristic result (CLAUDE.md LLM rule). When it fills the empty categories it must **not erase a level the heuristic was sure of** (a title token like "Senior") — so an LLM `unspecified` level falls back to the heuristic's level. (4) **`Classifier` port stays synchronous; `LLMClassifier` uses a sync `httpx.Client`** — deliberately no async ripple across the heuristic/backfill/tests. Justified because the LLM is a paying-customer API (not a polite crawl → does NOT route through `PoliteClient`/`Fetcher`) and classification runs inside the **sequential** per-posting ingest loop, so a blocking classify stalls nothing that could otherwise progress. (5) **Raw httpx to the Anthropic Messages API, no `anthropic` SDK dep** (matches the repo's thin-httpx style, e.g. `TelegramNotifier`, and keeps it fully `MockTransport`-testable): `POST /v1/messages` with `x-api-key`+`anthropic-version`, **JSON-only prompt** enumerating the Category/Level values, `max_tokens=256`, description clipped to 2000 chars for token cost.
